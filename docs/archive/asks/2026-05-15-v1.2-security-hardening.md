## 🔴 P0 — 立刻该做的（不阻断业务但风险确定）

### 1. **轮换 CF_API_TOKEN + 删除 hardcoded fallback**

**问题**：
- `public/_worker.js` line ~2179 + 我在 v1.1.4 加的 `/api/stream/upload-from-url` 都有 hardcoded fallback：
  ```js
  const CF_API_TOKEN = env.CF_API_TOKEN || ('cfut' + '_MGaGN86d75OIHOSPrVfqP7f2iw' + 'nhpxpbnSRhNNJi231d9654');
  ```
- 字符串拼接是为了绕 naive grep，但 token **已在 git 历史里多个 commit**。
- 任何拿到 repo 访问权的人（外包、未来开除的员工、潜在的 GitHub 漏洞）可拼接出这个 token。
- Token 权限：Stream:Edit + Stream:Read（可上传/删除/编辑 CF Stream 资源）。

**修复步骤**：

```bash
# 1. CF Dashboard → My Profile → API Tokens → Create Token
#    Template: Custom token
#    Permissions: Account → Stream → Edit
#    Resources: Include - Specific account - uvera (or All accounts)

# 2. wrangler 设新 secret
npx wrangler secret put CF_API_TOKEN
# 粘贴新 token,enter

# 3. 删两处 hardcoded fallback (public/_worker.js):
#    - line ~2179 (/api/stream/direct_upload)
#    - /api/stream/upload-from-url 我加的那段
# 改成: const CF_API_TOKEN = env.CF_API_TOKEN;
# 加 if (!CF_API_TOKEN) throw

# 4. CF Dashboard → 旧 token 详情 → Revoke

# 5. (可选,且谨慎) git history 重写删除旧 token
#    BFG: bfg --replace-text refs.txt
#    or git filter-repo (more modern)
#    ⚠️ 重写历史会影响所有 collaborator,需 force-push,需团队协商
#    建议除非有外部审计要求,直接接受历史泄露,保留 commit hash 完整性
```

**风险评估**：
- 不修：token 持续泄露,任何人能 access CF Stream 资源（删视频 / 上传垃圾内容刷 quota）。
- 修了：~30 min 工作量,zero 风险（fallback 删了后 Cloudflare secret 自动接管）。

### 2. **Server 端二次验证所有 tier-gated 字段**

来源：[`docs/decisions/2026-05-15-watermark-enforcement.md`](../../decisions/2026-05-15-watermark-enforcement.md) audit 清单。

`watermark` 字段曾经 client-trusted（v1.1.4 修了）。类似的同模式漏洞：

#### a) **`model` 字段**（中等风险）

`StoryGeneratorPage.jsx:543` 前端逻辑：
```js
if (profile.tier === 'free') {
  setVideoModel('ep-20260423195810-cx7nc');  // 锁 Fast
}
```

但 worker `/api/volcengine/video/submit:1252` 直接拿前端传的 `model` 参数。攻击：Free 用户改 payload 用 `ep-20260423195810-pro`（Seedance Pro 模型），免费用 Pro 模型。

**修复**：
```js
// worker /api/volcengine/video/submit
const FREE_LITE_ALLOWED_MODELS = ['ep-20260423195810-cx7nc'];  // Fast 路
if (isUnpaidTier && model && !FREE_LITE_ALLOWED_MODELS.includes(model)) {
  console.warn(`[video/submit] tier=${callerTier} sent unauthorized model=${model}, forcing Fast`);
  model = FREE_LITE_ALLOWED_MODELS[0];
}
```

#### b) **`resolution` 字段**（中等风险）

`StoryGeneratorPage.jsx:539` 前端用 `getResolutionOptions(tier)` 限制选项。Free 只能选 480p。

但 worker `/api/volcengine/video/submit:1252` 同样直接拿前端 `resolution`。攻击：Free 用户改 payload 传 `resolution: '1080p'`。

**修复**：
```js
// worker
import { getMaxResolution } from 'src/data/plans.js';  // (需要 worker 可访问的版本)
// or 复制 PLAN_LIMITS 到 worker
const maxAllowed = TIER_MAX_RESOLUTION[callerTier] || '480p';
const RESOLUTION_ORDER = ['480p', '720p', '1080p', '4K'];
if (RESOLUTION_ORDER.indexOf(resolution) > RESOLUTION_ORDER.indexOf(maxAllowed)) {
  console.warn(`[video/submit] tier=${callerTier} requested ${resolution} > max ${maxAllowed}, capping`);
  resolution = maxAllowed;
}
```

#### c) **`duration` 字段**（低风险）

不确定有 tier 上限。需要 audit `PLAN_LIMITS` 看是否有 duration 字段被前端限制但 worker 没复查。

#### d) **`generate_audio` 字段**（极低风险）

可能没 tier 差异。Free 可能不能用音频生成？要查 `PLAN_LIMITS` 是否标了。

## 🟡 P1 — v1.2.0 窗口内做

### 3. **`/api/admin/system-settings/update` 加 `stream_watermark_uid` 到允许列表**

目前 admin UI 的"Runtime configuration"卡片只能编辑 `lite_price_cooldown_hours`。worker `VALIDATORS` 没包含 `stream_watermark_uid`，所以 Leon 想在线换 watermark UID 改不了，必须 SQL 直接动表。

**修复**（`public/_worker.js` `/api/admin/system-settings/update` VALIDATORS）：
```js
const VALIDATORS = {
  lite_price_cooldown_hours: ...,
  stream_watermark_uid: (v) => {
    const s = String(v).trim();
    if (!/^[a-f0-9]{32}$/.test(s)) throw new Error('CF Stream UID 必须是 32 位小写 hex');
    return s;
  },
};
```

工作量：5 min。优先级 P1 因为 Leon 在 OPEN 留言里催过 admin 在线配置能力。

### 4. **撤掉 Service Worker 503 Offline fallback bug**

来源：earlier conversation. SW 没 fix。`public/sw.js` 在 worker fetch 失败时返回 plaintext "Offline"。已经在 v1.1.4 之前的 commit `00696fc` 替换成 real HTML 但需要 verify 真的修干净了 + 加测试。

### 5. **`generation_logs` stuck-row 自动清理 cron**

来源：v1.1.4 周期发现 28 条 non-video gen log 卡在 `status='started'` 因为 `random_ideas` / `concept_image` / `asset_describe` / `optimize_prompt` 这些 endpoint 的代码路径有 bug（**已修：loud-fail pattern**）。但**老 gen_type 的 stuck row 一直会累积**，建议 daily cron 自动清。

实现：每天 UTC 03:00 跑一次:
```sql
UPDATE generation_logs
SET status = 'failed',
    finished_at = now(),
    error_message = COALESCE(error_message, 'auto-marked failed: stuck >30min, daily cleanup cron')
WHERE status = 'started'
  AND started_at < now() - interval '30 minutes';
```

可用 Cloudflare Cron Triggers，也可用 GitHub Actions schedule。

## 🟢 P2 — 时间允许做

### 6. **抗滥用 rate limit**（无 auth 的 endpoint 当前可被滥用）

`/api/volcengine/video/submit` 注释里写"legacy decision",没 gate。攻击：匿名脚本疯狂 POST 烧 ARK quota。

可以：
- 加 rate limit per IP (Cloudflare Workers + KV / Durable Objects)
- 或：要求所有 video gen 必须 authenticated

### 7. **审计 `/auth/v1/admin/users/{id}` 使用频率**

worker 在多个 endpoint 里频繁 fetch user metadata（webhook、reconcile、video/submit、stream upload...）。Supabase Auth Admin API 有 rate limit。如果用户量上来可能撞顶。

可以：cache user metadata in module-level Map with TTL (类似 `_systemSettingsCache` 模式)。

### 8. **Webhook signature replay attack 防护**

Stripe webhook 当前只验签名,没验 timestamp 时间窗（防 replay）。规范是 5 min。

Stripe SDK 自动处理,但我们手写的 HMAC 验证没加。看 [`public/_worker.js`](../../../public/_worker.js) line ~3823 的签名验证段。

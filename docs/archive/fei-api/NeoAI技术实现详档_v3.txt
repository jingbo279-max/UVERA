0. 公共说明
0.1 后端基础地址
系统通过管理后台 app_settings 的 api_env 字段动态切换两套环境：
# 开发环境
API_BASE_DEV  = "https://dev.neodomain.cn"
# 生产环境（默认）
API_BASE_PROD = "https://story.neodomain.cn"
# AI 模型网关（编剧 / Vision）
AI_API_BASE   = "https://ai-api.neodomain.cn/v1"
0.2 MemfireDB Edge Functions
前端通过 MemfireDB 函数计算调用所有边缘函数，函数名映射为去连字符的 kebab-less 形式：
MEMFIRE_FN_BASE = "https://functions5.memfiredb.com/d6vussgg91hj59690tv0"
 
ai-screenwriter   →  /aiscreenwriter
concept-design    →  /conceptdesign
video-generate    →  /videogenerate
oss-upload        →  /ossupload
save-work         →  /savework
user-auth         →  /userauth
0.3 通用请求头
Content-Type:  application/json
Authorization: Bearer <MEMFIRE_ANON_KEY>      # 调用 Edge Function 时
apikey:        <MEMFIRE_ANON_KEY>             # 调用 Edge Function 时
accessToken:   <用户登录 token>                # 调用 neodomain 业务接口时

{

  "errCode":  "" ,            // 失败时返回错误码
  "errMessage": "",           // 失败时返回中文描述
  "data": { /* 业务数据 */ }

1. 功能一：创建角色
1.1 调用链路
前端
  ├─ POST  /agent/sts/credentials              → 换取阿里云 OSS 临时凭证
  ├─ PUT   <ossEndpoint>/<bucket>/<key>        → 直传照片到 OSS（上海节点）
  └─ POST  Edge: /userauth                     → 写 user_characters 记录
1.2 步骤 1：换取 STS 临时凭证
请求
POST  {API_BASE}/agent/sts/credentials
Headers:
  Content-Type: application/json
  accessToken:  <用户登录 token>
Body: {}
响应示例
{
  "success": true,
  "data": {
    "accessKeyId":     "STS.NTxxxx",
    "accessKeySecret": "wJalrxxxx",
    "securityToken":   "CAIS+gF1q6Ft5xxxx",
    "expiration":      "2026-04-18T17:30:00Z",
    "endpoint":        "oss-cn-shanghai.aliyuncs.com",
    "bucket":          "neoai-prod",
    "region":          "cn-shanghai"
  }
}
1.3 步骤 2：直传照片到 OSS
PUT https://<bucket>.<endpoint>/characters/<userId>/<uuid>.jpg
Headers:
  Authorization:        OSS <accessKeyId>:<signature>   # 双签名重试：STS 优先，AK/SK 兜底
  x-oss-security-token: <securityToken>
  Content-Type:         image/jpeg
Body: <二进制照片>
 
→ 成功后得到 photoUrl:
   https://neoai-prod.oss-cn-shanghai.aliyuncs.com/characters/u-123/9f8e.jpg
失败时按"STS → AK/SK"双签名顺序重试，最多 2 次。
1.4 步骤 3：（可选）Vision 提取身份特征
在生成剧本时由 ai-screenwriter 函数自动调用一次，前端不需要单独调用。提示词如下：
POST {AI_API_BASE}/chat/completions
Headers:
  Authorization: Bearer <NEODOMAIN_API_KEY>
  Content-Type:  application/json
Body:
{
  "model": "gemini-2.5-flash-lite",
  "messages": [{
    "role": "user",
    "content": [
      { "type": "image_url", "image_url": { "url": "data:image/jpeg;base64,<...>" }},
      { "type": "text",      "text":      "请用中文简要描述这张照片中人物的身份特征，包括：\n1. 性别（男/女）\n2. 大致年龄段\n3. 外貌特征\n4. 穿着风格\n\n只输出描述文字。" }
    ]
  }]
}
Vision 输出示例
"年轻男性，大约 25-30 岁，短发偏黑，方脸轮廓，浓眉，皮肤偏白，
身着深灰色连帽卫衣，整体气质沉稳干净，有都市青年感。"

1.6 错误码
错误码
含义 / 处理建议
NO_FACE_DETECTED
未识别到人脸 → 引导用户重拍，提示光线 / 取景
UPLOAD_FAILED

2. 功能二：生成剧本
2.1 调用链路
前端 → Edge: /aiscreenwriter
                 ├─ Vision 抽取身份特征（gemini-2.5-flash-lite）
                 └─ 编剧主调用（gemini-3-flash-preview）→ JSON 结构化剧本
2.2 请求
POST  {MEMFIRE_FN_BASE}/aiscreenwriter
Headers:
  Authorization: Bearer <MEMFIRE_ANON_KEY>
  Content-Type:  application/json
Body:
{
  "style":        "wes-anderson",          // 风格 ID
  "styleName":    "韦斯·安德森",            // 风格中文名（用于查 prompt 模板）
  "transcript":   "一个咖啡师在午后阳光下煮一杯特别的咖啡，最后抬头微笑",
  "videoType":    "trailer",               // trailer / vlog / mv / tvc / promo / short-drama
  "hasPhoto":     true,
  "photoData":    "data:image/jpeg;base64,<...>",  // base64 或 OSS URL
  "previousStory": null,                   // 接龙创作时为上段剧情摘要
  "apiBase":      "https://story.neodomain.cn"
}
2.3 关键 Prompt 模板
System Prompt（节选，按 videoType 动态拼接）
你是一位{role}，同时也是 Seedance 2.0 视频提示词大师。
 
## 核心任务
根据用户提供的动画风格和故事描述，创作一个{videoTypeLabel}风格的多镜头视频脚本。
 
## 视频类型叙事指导
- 叙事结构：{structure}
- 整体调性：{tone}
- 对白风格：{dialogueStyle}
 
## ⚠️ 时长硬性约束（最高优先级）
- 总时长必须在 8-15 秒之间
- 设计 2-4 个镜头，每个镜头 2-6 秒
- 镜头时长必须有节奏变化（如 3+5+2）
- 所有镜头时长之和 = totalDuration
 
## 一、镜头设计规范
每个镜头必须包含五层描述：
1. 运镜（摄影术语+方向+速度）
2. 角色动作（具象化拆解，禁止模糊）
3. 面部 / 口型（表情+嘴部+眼神）
4. 场景环境（空间、材质、植被、天气、光影、道具、动态元素）
5. 风格锚定 + 时间分配（前 2 秒...第 3-5 秒...）
 
## 四、输出格式（严格 JSON，不要 markdown）
{
  "totalDuration": 10,
  "segments":      [{"tag":"镜头1","text":"..."}],
  "shots":         [{"shotNumber":1,"duration":"5秒","camera":"...","action":"...",
                     "lighting":"...","sound":"...","dialogue":"...","videoPrompt":"≥80字"}],
  "music":         {"style":"...","mood":"...","reference":"..."},
  "videoPrompt":   "整合所有镜头的完整中文提示词",
  "mood":          "整体情绪氛围",
  "summary":       "一句话 tagline"
}
User Prompt（动态拼接）
动画风格：韦斯·安德森
 
## 风格视觉指导
画面极致对称构图，柔和糖果色调（粉/黄/薄荷绿），微缩景观模型感，如精致的定格动画布景。
预设情绪基调：冷幽默、精致、怀旧
推荐配乐方向：60 年代法国流行乐或英伦民谣，轻快而略带忧郁
 
用户故事描述：一个咖啡师在午后阳光下煮一杯特别的咖啡，最后抬头微笑
 
⚠️ 重要：用户已上传了自己的照片作为角色参考。
 
📸 照片人物身份识别结果：年轻男性，约 28 岁，短发偏黑，方脸轮廓，
身着深灰色连帽卫衣，整体气质沉稳干净。
 
你的剧本主角必须严格匹配以上身份特征。每个镜头都必须有这个角色出镜。
 
⚠️ 最重要的约束：
- 总时长 8-15 秒
- 2-4 个镜头，每镜头 2-6 秒
- 每个 videoPrompt 必须 ≥80 字，且包含精确时间段内容分配
- 严禁出现任何文字 / 字幕 / LOGO / 水印

3. 功能三：选择风格
3.1 说明
风格选择是纯前端 + 配置层的操作：所有 52 种风格的元数据（中文名、英文 prompt、服装提示词、配乐建议、是否需要真人审核）都已在 ai-screenwriter 与 concept-design 函数内固化为 styleScreenwritingMap / styleMap 常量。前端只负责把 styleId 与 styleName 透传给后续两个函数。
3.2 风格元数据结构
styleScreenwritingMap[styleName] = {
  visualGuidance: "画面极致对称构图，柔和糖果色调...",   // 注入编剧 prompt
  moodPreset:     "冷幽默、精致、怀旧",                  // 推荐情绪
  musicPreset:    "60 年代法国流行乐..."                 // 推荐配乐
}
 
styleMap[styleName] = {
  en:      "Wes Anderson aesthetic, perfectly symmetrical composition...",  // 注入图片 prompt
  costume: "vintage tailored suit or corduroy outfit, bow tie..."           // 推荐服装
}
3.3 触发真人审核的风格集合
REAL_PERSON_REVIEW_STYLES = new Set([
  'souls-dark-fantasy',          // 黑魂式暗黑幻想
  'gritty-realism',              // 硬核工业写实
  'painterly-npr',               // 唯美手绘 NPR
  'vhs-horror',                  // 录像带模拟恐怖
  'cinematic-photorealistic',    // 写实电影感
  'illumination',                // 照明娱乐风
  'wes-anderson',                // 韦斯·安德森
  'belleville',                  // 佳丽村三姐妹
  'scanner-darkly',              // 黑暗扫描风
])
3.4 前端选择后写入 works 表
UPDATE works
SET style       = 'wes-anderson',
    style_name  = '韦斯·安德森'
WHERE id = '<workId>';

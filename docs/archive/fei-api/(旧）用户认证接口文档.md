# 用户认证接口文档

## 1. 统一发送验证码接口

### 接口信息
- **接口路径**: `https://story.neodomain.cn/user/login/send-unified-code`
- **请求方式**: `POST`
- **接口描述**: 统一发送验证码接口,支持手机号和邮箱两种方式

### 请求参数

#### Headers
无特殊要求

#### Body (JSON)
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| contact | String | 是 | 手机号或邮箱地址 | `13800138000` 或 `user@example.com` |

#### 请求示例
```json
{
  "contact": "13800138000"
}
```

或

```json
{
  "contact": "user@example.com"
}
```

### 响应参数

#### 成功响应
```json
{
  "success": true,
  "errCode": null,
  "errMessage": null
}
```

#### 失败响应
```json
{
  "success": false,
  "errCode": "PARAM_ERROR",
  "errMessage": "请输入有效的手机号或邮箱地址"
}
```

### 参数校验规则
- **手机号格式**: 以1开头,第二位为3-9,共11位数字 (正则: `^1[3-9]\d{9}$`)
- **邮箱格式**: 标准邮箱格式 (正则: `^[a-zA-Z0-9_-]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$`)

## 2. 统一登录接口

### 接口信息
- **接口路径**: `https://story.neodomain.cn/user/login/unified-login`
- **请求方式**: `POST`
- **接口描述**: 统一登录接口,支持手机号和邮箱验证码登录

### 请求参数

#### Headers
无特殊要求

#### Body (JSON)
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| contact | String | 是 | 手机号或邮箱地址 | `13800138000` 或 `user@example.com` |
| code | String | 是 | 验证码(6位) | `123456` |
| invitationCode | String | 否 | 邀请码 | `INVITE123` |

#### 请求示例
```json
{
  "contact": "13800138000",
  "code": "123456"
}
```

或

```json
{
  "contact": "user@example.com",
  "code": "123456",
  "invitationCode": "INVITE123"
}
```

### 响应参数

#### 成功响应
```json
{
  "success": true,
  "data": {
    "authorization": "eyJhbGciOiJIUzUxMiJ9...",
    "userId": "1234567890",
    "email": "test@example.com",
    "mobile": "138****8000",
    "nickname": "John Doe",
    "avatar": "https://example.com/avatar.jpg",
    "status": 1
  },
  "errCode": null,
  "errMessage": null
}
```

#### 失败响应
```json
{
  "success": false,
  "data": null,
  "errCode": "PARAM_ERROR",
  "errMessage": "请输入有效的手机号或邮箱地址"
}
```

### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| authorization | String | JWT认证令牌 |
| userId | String | 用户ID |
| email | String | 用户邮箱 |
| mobile | String | 用户手机号(已脱敏) |
| nickname | String | 用户昵称 |
| avatar | String | 用户头像URL |
| status | Integer | 用户状态: 0-未激活, 1-正常, 2-禁用 |

### 参数校验规则
- **contact**: 不能为空,必须是有效的手机号或邮箱
- **code**: 不能为空,长度必须为6位
- **invitationCode**: 可选参数



## 通用说明

### 错误码说明
| 错误码 | 说明 |
|--------|------|
| PARAM_ERROR | 参数错误 |
| SEND_FAILED | 验证码发送失败 |
| USER_ALREADY_EXISTS | 用户已存在 |
| VERIFY_CODE_ERROR | 验证码错误 |
| USER_NOT_FOUND | 用户不存在 |

### 注意事项
1. 所有接口的 `Content-Type` 均为 `application/json`
2. 验证码有效期通常为5-10分钟(具体由后端配置决定)
3. 手机号会在响应中进行脱敏处理(如: `138****8000`)
4. JWT Token 需要在后续请求中通过 `Authorization` 请求头携带
5. 接口路径前缀为 `/user/login/`,完整路径需要加上应用的base path(如: `/agent/v1/user/login/`)
